Modelos de Concurrencia

1 / 16

El Problema: Tareas Lentas

Muchas tareas, como esperar una respuesta de red o un cálculo complejo, "bloquean" el programa, impidiendo que haga otras cosas.

2 / 16

Concurrencia vs. Paralelismo

Concurrencia: Gestionar múltiples tareas a la vez (un chef cortando verduras mientras vigila una olla).

Paralelismo: Ejecutar múltiples tareas a la vez (varios chefs cocinando al mismo tiempo).

3 / 16

El Gran Obstáculo: El GIL

El Global Interpreter Lock es un cerrojo en CPython que solo permite que un hilo ejecute bytecode de Python a la vez. Esto significa que el `threading` no logra paralelismo real en tareas de CPU.

4 / 16

Modelo 1: threading

Ejecuta tareas en hilos separados dentro del mismo proceso. Comparten la misma memoria.

5 / 16

¿Cuándo usar `threading`?

Ideal para tareas I/O-bound (limitadas por Entrada/Salida). Mientras un hilo espera por la red o el disco, el GIL se libera y otro hilo puede ejecutarse.

# Ejemplos I/O-bound:
# - Peticiones a APIs
# - Lectura/escritura de archivos
# - Consultas a bases de datos
6 / 16

Ejemplo con threading

Ambas tareas "esperan" concurrentemente. El tiempo total es de ~2s, no 3s.

import threading
import time

def tarea_io(nombre, duracion):
    print(f"{nombre} esperando...")
    time.sleep(duracion)
    print(f"{nombre} terminó.")

t1 = threading.Thread(target=tarea_io, args=("A", 1))
t2 = threading.Thread(target=tarea_io, args=("B", 2))
t1.start()
t2.start()
t1.join()
t2.join()
7 / 16

Modelo 2: multiprocessing

Ejecuta tareas en procesos completamente separados. Cada proceso tiene su propio intérprete de Python, su propia memoria y su propio GIL.

8 / 16

¿Cuándo usar `multiprocessing`?

Ideal para tareas CPU-bound (limitadas por el procesador). Permite el paralelismo real, aprovechando todos los núcleos de la CPU.

# Ejemplos CPU-bound:
# - Cálculos matemáticos intensivos
# - Procesamiento de imágenes o video
# - Simulaciones científicas
9 / 16

Ejemplo con multiprocessing

Ambas tareas se ejecutan en paralelo en diferentes núcleos de CPU.

from multiprocessing import Process

def tarea_cpu(nombre):
    print(f"Proceso {nombre} iniciado.")
    # Simula un cálculo intensivo
    [i*i for i in range(10**7)]
    print(f"Proceso {nombre} terminado.")

p1 = Process(target=tarea_cpu, args=("A",))
p2 = Process(target=tarea_cpu, args=("B",))
p1.start()
p2.start()
p1.join()
p2.join()
10 / 16

Modelo 3: asyncio

Concurrencia en un solo hilo. Un "event loop" gestiona las tareas, que ceden el control explícitamente con `await` cuando están esperando.

11 / 16

¿Cuándo usar `asyncio`?

Ideal para tareas I/O-bound de alto volumen. Es extremadamente eficiente para manejar miles de conexiones de red simultáneas (ej. en un servidor web).

12 / 16

Ejemplo con asyncio

El tiempo total es de ~2s, no 3s. `asyncio.sleep` simula una espera de red no bloqueante.

import asyncio

async def tarea_async(nombre, duracion):
    print(f"{nombre} esperando...")
    await asyncio.sleep(duracion)
    print(f"{nombre} terminó.")

async def main():
    await asyncio.gather(
        tarea_async("A", 1),
        tarea_async("B", 2)
    )
asyncio.run(main())
13 / 16

Resumen Comparativo

Threading: Concurrencia, I/O-bound, memoria compartida.

Multiprocessing: Paralelismo, CPU-bound, memoria separada.

Asyncio: Concurrencia en un hilo, I/O-bound de alto volumen.

14 / 16

Regla de Oro: ¿Cuál Elegir?

¿Tu programa está "pensando" mucho?
Usa multiprocessing.

¿Tu programa está "esperando" mucho?
Usa threading (para empezar) o asyncio (para aplicaciones de red avanzadas).

Conclusión

Python ofrece un modelo de concurrencia para cada tipo de problema. Elegir el correcto es clave para escribir aplicaciones rápidas y eficientes.
